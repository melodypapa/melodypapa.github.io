<section id="title">ARM数据存储的大小端</section>

ARM 体系机构可以使用两个不同的方式存储字数据，即：大端格式和小端格式。

**注意：**

系统初始化时末日为小端格式。

# 1. 大端格式

在大端格式中，32位的字数据的高字节保存在低地址中，而字数据的低字节则保存在高地址中。

假如现有一32位int型数0x12345678，那么其MSB(Most Significant Byte，最高有效字节)为0x12，其LSB (Least Significant Byte，最低有效字节)为0x78，在CPU内存中有两种存放方式：（假设从地址0x4000_0000开始存放）

| Address    | Data |
| ---------- | ---- |
| 04000_0000 | 0x12 |
| 04000_0001 | 0x34 |
| 04000_0002 | 0x56 |
| 04000_0003 | 0x78 |

# 2. 小端格式

在小端格式中，32位的字数据的高字节保存在高地址中，而字数据的低字节则保存在地址中。

假如现有一32位int型数0x12345678，那么其MSB(Most Significant Byte，最高有效字节)为0x12，其LSB (Least Significant Byte，最低有效字节)为0x78，在CPU内存中有两种存放方式：（假设从地址0x4000_0000开始存放）

| Address    | Data |
| ---------- | ---- |
| 04000_0000 | 0x78 |
| 04000_0001 | 0x56 |
| 04000_0002 | 0x34 |
| 04000_0003 | 0x12 |

# 3. 字节对齐

## 早期的ARM7和ARM9

对于早期的 **ARM7** 和 **ARM9** 微处理器支持字节（8位）、半字（16位）、字（32位）三种数据类型。其中字需要4字节对齐，即地址的低两位为0；半字需要2字节对齐，即地址的最低位为0。

同时每一种数据类型分别支持带符号数和无符号数，所以共支持6种数据类型：

1. 带符号的字节类型
2. 无符号的字节类型
3. 带符号的半字类型
4. 无符号的半字类型
5. 带符号的字类型
6. 无符号的字类型

除了数据传送指令支持较短的字节和半字的数据类型外，在ARM内部所有的操作都是面向32位操作数的。

## Cortext-M

和 **ARM7** 和 **ARM9** 不同，Cortex-M处理器中的 **16** 位和 **32** 位的数据项不必与字长或者半字长边界对齐。Cortex-M的处理器可以进行未对齐内存访问，这特性使编译器和链接器能更好的利用 **SRAM**。
